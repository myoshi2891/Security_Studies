# ソフトウェア開発セキュリティ完全ガイド 2026

2026年3月時点の最新脅威ランドスケープに基づき、**SDLC・OWASP・AI・サプライチェーン・PQC**の各カテゴリーを初学者にもわかりやすくステップバイステップで解説します。

---

## Section 01 — 2026年の脅威ランドスケープ

なぜ今セキュリティが最重要課題なのか？産業化されたサイバー犯罪の実態を理解する必要があります。

### 主要な4つの潮流
- **AI兵器化の加速**: LLMをフォースマルチプライヤーとして悪用。PROMPTFLUXがEDRパターンをリアルタイム学習・回避。WEF調査で87%がAI関連脆弱性を最大リスクと認識。
- **アイデンティティが新境界**: 攻撃者は「侵入」せず正規認証で「ログイン」。初期アクセスブローカー間のアクセス権移転は30秒未満。OAuthトークン・セッションCookie窃取が横行。
- **産業化されたサプライチェーン攻撃**: SaaSベンダー経由でハードコードされたキーやPATを奪い顧客環境へ水平展開。単独ハッカーから高度分業組織へ完全移行。
- **経営陣の個人責任**: 重大過失による漏洩で経営陣が個人的法的責任を問われるリスク増大。セキュリティが取締役会レベルの四半期デューデリジェンス事項に昇格。

### 2026年 主要サイバーセキュリティトレンド
| トレンド | ソフトウェア開発への影響 |
| :--- | :--- |
| **Agentic AI（自律型AI）の台頭** | 人間不介入で偵察・横展開・悪用を実行。AI主導の検知・自動修復をコードレベルで組み込む必要あり。 |
| **ディープフェイク・アイデンティティ欺瞞** | リアルタイムディープフェイクで生体認証をバイパス。パスキー+継続的セッション検証への移行が必須。 |
| **セキュア・バイ・デザインの義務化** | 設計段階からの脅威モデリングが標準要件化。リリース後のパッチ当てモデルは完全に破綻。 |
| **Q-Day（量子コンピューター）** | 今収集・後で解読の攻撃が進行中。NISTのPQC標準（FIPS 203/204/205）への早急な移行計画が必要。 |

## Section 02 — Secure SDLC ─ "シフトエブリウェア"

従来の「シフトレフト」からライフサイクル全体で継続検証する「シフトエブリウェア」へ。

### ステップバイステップ実践ガイド
1.  **要件定義・設計フェーズ ─ 脅威モデリング (STRIDE)**: 機能要件と同時にセキュリティ要件を文書化し、脅威モデリングを実施します。STRIDEの6カテゴリでアーキテクチャの欠陥を洗い出します。ApiiroなどAI駆動型ツールでの自動化が進んでいます。
2.  **セキュアコーディング ─ CI/CDへのSAST/SCA統合**: コミット時にSAST（静的解析）でSQLi・XSSを検出し、SCA（依存関係解析）で既知CVEをスキャン。GHAS・Semgrep・Veracodeが標準です。
3.  **デプロイメント保護 ─ DAST・IaCスキャン**: 稼働中アプリにDAST（動的テスト）で認証バイパスを検出。Terraform・K8sの設定ミスも自動スキャン（Trivy等）で防止。
4.  **本番環境の継続監視 ─ 測定可能な改善（NIST SSDF PO.6）**: WAF等でフロントドアを保護。NIST SSDF 1.2は実際のシグナルに基づく「生きた改善バックログ」の維持を要求しています。

### STRIDEチートシート
- **S — Spoofing（なりすまし）**: MFA、OAuth 2.0、mTLS（SPIFFE/SPIRE）
- **T — Tampering（改ざん）**: AES-256暗号化、デジタル署名、コミット署名
- **R — Repudiation（否認）**: イミュータブル監査ログ、WORMストレージ
- **I — Info. Disclosure（情報漏洩）**: 行レベルセキュリティ、メモリ分離、シークレットスキャン
- **D — Denial of Service**: APIレートリミット、意味論的レート制限、オートスケール
- **E — Elevation of Privilege**: 最小権限原則、RBAC、コンテナroot権限剥奪

## Section 04 — LLM / AI エージェントのセキュリティ

2026年、AIは「自律的に行動する存在（Agentic AI）」へ進化。

### 主要脅威
- **LLM01: Prompt Injection**: 間接的プロンプトインジェクション。PDFやWebページに不可視の命令を潜ませAIを乗っ取る。
- **LLM06: Excessive Agency**: AIエージェントへの過剰な機能・権限・自律性の付与。最小特権原則 + Human-in-the-loopの設計が必須。
- **LLM07: System Prompt Leakage**: システムプロンプトはセキュリティ境界ではない。APIキーを含めた瞬間「失われた」と見なすべき。

## Section 05 — AIコーディングアシスタントの安全な利用

38%の企業がAIツール関連インシデントを経験。

- **Slopsquatting**: AIが実在しないパッケージ名を幻覚で生成。npm installで即侵害。
- **コンテキスト漏洩**: `.env` や秘密鍵がAI推論コンテキストとしてクラウドへ送信されるリスク。Content Exclusion設定必須。
- **IDEプロンプトインジェクション**: README内の不可視テキストによるLOTAIL攻撃。

## Section 06 — ソフトウェアサプライチェーンセキュリティ

SBOM・SLSA・VEXによる法的義務化。

- **SBOM (Software Bill of Materials)**: 全コンポーネントのリスト化。CycloneDX / SPDXフォーマット。
- **SLSA (Supply-chain Levels for Software Artifacts)**: ビルドプロセスの完全性を暗号学的に保証。
- **EU CRA (Cyber Resilience Act)**: 2026年9月から報告義務開始。非準拠はEU市場で販売禁止リスク。

## Section 07 — 耐量子暗号 (PQC)

"Harvest now, decrypt later" 脅威への備え。

- **NIST標準**: FIPS 203 (ML-KEM), 204 (ML-DSA), 205 (SLH-DSA)。
- **Crypto-Agility**: アルゴリズムをハードコードせず、設定で動的に切り替えられる抽象化レイヤーの設計。
- **2035年目標**: 米国国家安全保障システム（NSS）の完全耐量子化。
