# 脅威ランドスケープ 2026

サイバー攻撃は「孤独なハッカー」の時代を完全に終え、**高度に産業化された分業型犯罪エコシステム**へと変貌しました。本ガイドでは、2026年3月時点の最新の一次ソース（M-Trends, WEF, PwC等）に基づき、現代のエンジニアが直面している「生きた地図」を詳細に解説します。

## Section 01 — 脅威ランドスケープとは何か？

**脅威ランドスケープ（Threat Landscape）**とは、ある特定の時点において、組織やシステムに対して存在するすべての脅威・攻撃者・攻撃手法・脆弱性の全体像を指します。

### 構成する4つの要素
1.  **脅威アクター (Threat Actors)**: 攻撃を行う主体（国家、犯罪組織、内部犯）。
2.  **攻撃ベクター (Attack Vectors)**: システムへの侵入経路（フィッシング、サプライチェーン）。
3.  **脆弱性 (Vulnerabilities)**: システムの弱点（未パッチのコード、設定ミス）。
4.  **インパクト (Impact)**: 攻撃成功時の被害（データ漏洩、業務停止、法的責任）。

## Section 02 — 2026年の支配的なパラダイム：産業化されたサイバー犯罪

2026年のサイバー攻撃における最大のパラダイムシフトは、攻撃の「産業化」と「分業化」です。

### 攻撃者間の「30秒の連携」
M-Trends 2026の衝撃的なデータによれば、**初期アクセスブローカー（IAB）がシステムへの初期アクセスを獲得してから、別のアクター（ランサムウェアオペレーター等）にそのアクセス権を譲渡・売却するまでの時間は30秒未満**にまで短縮されています。

| 役割 | 専門業務 | 目的 |
| :--- | :--- | :--- |
| **初期アクセスブローカー (IAB)** | 脆弱性探索、認証情報の窃取、初期侵入 | 侵入経路の確保と販売 |
| **ランサムウェアオペレーター** | データの暗号化、バックアップ破壊、身代金要求 | 金銭的利益（二重恐喝） |
| **データ流出スペシャリスト** | 重要データの選別、高速な外部転送 | データの窃取とゆすり |

## Section 03 — 主要な脅威アクター（2026年版）

### 1. 国家支援型アクター (Nation-State Actors)
- **特徴**: 予算・技術・時間が潤沢。
- **2026年の動向**: 「Harvest now, decrypt later」攻撃を組織的に実行。PQC（耐量子計算機暗号）導入前のデータを将来の解読のために蓄積。

### 2. 組織的サイバー犯罪グループ (Cybercrime Groups)
- **特徴**: 利益最大化を追求。RaaS（Ransomware-as-a-Service）を運営。
- **2026年の動向**: AIによる攻撃自動化。企業の株価やM&A情報を狙った「経済的恐喝」へシフト。

### 3. 初期アクセスブローカー (IAB)
- **特徴**: 侵入の「窓口」を作るプロフェッショナル。
- **2026年の動向**: 24時間365日の自動スキャンにより、公開された脆弱性を数分以内に特定。

## Section 04 — AIの兵器化：フォースマルチプライヤー

AIは攻撃者にとって、攻撃の質と量を飛躍的に向上させる「フォースマルチプライヤー（戦力乗数）」となっています。

- **超精密フィッシング**: ターゲットのSNSや公開情報を学習し、完璧に自然な文章・文脈でメールを生成。
- **リアルタイムディープフェイク**: 役員や上司の「声」と「顔」をリアルタイムで合成し、Web会議や電話で不正送金を指示。
- **PROMPTSTEAL / PROMPTFLUX**: AIシステム自体のプロンプトからAPIキーや内部ロジックを抽出する攻撃ツールの一般化。

> **AIは「双方向の剣」**: 防御側にとってもAIは最強の武器（SAST/DAST/EDRの高度化）ですが、攻撃者の進化スピードに追随することが2026年の必須条件です。

## Section 05 — アイデンティティが新たな境界線

クラウド・SaaS・リモートワークの普及により、従来の「社内ネットワークを守る境界防御」は完全に崩壊しました。

### 攻撃手法のシフト
攻撃者は今や「ネットワークに侵入」するのではなく、**正規の認証情報を使って「ログイン」**します。

- **フィッシング（最多発生）**: AIによる完璧な偽サイト誘導。AiTM攻撃によるMFA突破。
- **クレデンシャル・スタッファリング**: 漏洩リストを用いた自動ログイン試行。
- **セッションCookie / OAuthトークン窃取**: 認証後のセッション情報を直接盗む（Evilginx等の利用）。
- **AIによる生体認証突破**: 顔・声のディープフェイクによる認証バイパス。

### ゼロトラストの3原則 (Always Verify)
1.  **常に検証する (Always Verify)**: 位置に関係なく、すべてのアクセスを認証・認可。
2.  **最小権限の付与 (Least Privilege)**: 必要なとき、必要な分だけ（JIT）。
3.  **侵害を前提とする (Assume Breach)**: 横展開（ラテラルムーブメント）を封じる設計。

## Section 06 — 攻撃速度の激変 ─ 30秒で何が起きるか

ドウェルタイム（侵入から検知までの潜伏期間）は、かつての平均200日から、2026年には数時間〜数日へと劇的に短縮されています。

### モダン攻撃のキルチェーン
1.  **偵察** → 2. **初期侵入** → 3. **永続化** → 4. **アクセス権売却（30秒以内）** → 5. **横展開** → 6. **データ収集** → 7. **実行・流出**

## Section 07 — 規制の強化 ─ セキュリティが法的責任に

2026年、セキュリティは「努力目標」から「法的義務」になりました。

- **EU Cyber Resilience Act (CRA)**: 2026年9月から脆弱性・インシデントの報告義務開始。
- **経営陣の個人的責任**: 重大過失による漏洩時、CISOやCEOが個人として法的責任を問われるケースが増大。
- **NIST SSDF v1.2**: 開発プロセス全体の証明が政府調達の必須要件。

## Section 08 — 初学者が今すぐ始めるべき7ステップ

1.  **パスキー (FIDO2)** でMFAをアップグレード。
2.  **シークレットをコードに含めない**（gitleaks等の導入）。
3.  **依存関係の脆弱性スキャン (SCA)** をCI/CDに統合。
4.  **入力値検証**（SQLi, XSS対策）の徹底。
5.  **エラーメッセージのサニタイズ**（内部情報の隠蔽）。
6.  **依存関係の出所を確認**（Slopsquatting対策）。
7.  **脅威インテリジェンスの継続的学習**（M-Trends等の購読）。
