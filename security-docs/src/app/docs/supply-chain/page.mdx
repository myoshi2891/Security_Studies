<SectionCard
  eyebrow="Section 03 — SBOM & VEX"
  title="透明性の確保：SBOMとVEX"
>
  <HighlightBox color="blue">
    **SBOM (Software Bill of Materials)** とは、ソフトウェアに含まれるすべての部品を網羅した「原材料リスト」です。
  </HighlightBox>

  <DataTable
    headers={["技術", "フルネーム", "役割"]}
    rows={[
      ["SPDX", "Software Package Data Exchange", "ISO標準のSBOM形式"],
      ["CycloneDX", "Cyclone Data Exchange", "軽量でセキュリティに特化したSBOM形式"],
      ["VEX", "Vulnerability Exploitability eXchange", "脆弱性が実際に「悪用可能か」を示す対の文書"]
    ]}
  />

  <Callout type="info" title="VEXの重要性">
    SBOMだけでは「大量の脆弱性リスト」に圧倒されます。VEXを使うことで「この脆弱性は含まれるが、設定により悪用不可である」ことを伝え、不必要な調査を減らせます。
  </Callout>
</SectionCard>

    <HighlightBox color="cyan">
      **Cosign**
      Sigstoreプロジェクトのツール。コンテナイメージやSBOMに暗号署名を付与し、改ざんを検知。
    </HighlightBox>
  </div>

  <Terminal title="cosign-verify.sh">
